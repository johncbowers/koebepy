# PlanarMap to koebepy DCEL State Pipeline

**Convert PlanarMap-generated triangulations into koebepy DCEL state JSON files.**

This folder contains a Python pipeline that bridges PlanarMap (an external tool by Gilles Schaeffer) with koebepy's DCEL representation. The original PlanarMap source is kept external; only the bridge code is committed here. (Note that the Original Planar Map code uses deprecated lrand() and srand() functions)

**Attribution:**
- PlanarMap by Gilles Schaeffer
- https://www.lix.polytechnique.fr/~schaeffe/PagesWeb/PlanarMap/index-en.html

---

## Overview

The pipeline converts PlanarMap-derived triangulations into koebepy DCEL state JSON files via this end-to-end flow:

1. Generate/parse triangulations with `triangulation_pipeline_scaffold.py`
2. Convert DCEL topology into koebepy state format with `koebepy_dcel_state_exporter.py`
3. Organize results into a test suite with `batch_dcel_state_generator.py`

### Output Format

The exported JSON uses the same state keys as koebepy DCEL serialization:
- `vert_data`: list of `[aDartIdx, vertexData]`
- `dart_data`: list of dictionaries with edge/origin/face/prev/next/twin/data
- `edge_data`: list of `[aDartIdx, edgeData]`
- `face_data`: list of `[aDartIdx, faceData]`
- `outer_face_idx`: outer face index or null

## Setup

### Prerequisites

- Python 3.8+
- External PlanarMap installation (install separately; not included)

### Environment Variables

Set one of the following to locate PlanarMap:
- `PLANARMAP_BIN`: full path to `planarmap.exe`
- `PLANARMAP_ROOT`: folder containing `planarmap.exe`
- Default fallback: `../planarmap.exe` (relative to this folder)

Example:
```powershell
$env:PLANARMAP_BIN = "C:\path\to\planarmap.exe"
```

## Quick Start

### Single File Conversion

Convert an existing DCEL JSON to koebepy state JSON:

```bash
python -c "from koebepy_dcel_state_exporter import export_dcel_state_from_json_file; export_dcel_state_from_json_file('one_map.json','dcel_examples/one_map_state.json'); print('ok')"
```

Inspect the output:
```bash
type dcel_examples/one_map_state.json
```

### Batch Generation

Generate 20 random triangulations, convert to DCEL state JSON, and organize into a test suite:

```bash
python batch_dcel_state_generator.py --count 20 -C2 -F100 -d
```

The script forwards unknown CLI flags directly to `planarmap.exe`.

Important: `-n` is not a PlanarMap size flag. Use PlanarMap size/family flags such as:
- `-V<number>`: number of vertices
- `-F<number>`: number of faces
- map family flags like `-C2`, `-M2`, etc.
- `-d` to dualize output when needed for triangulation workflows

### Verified PlanarMap Flags (`planarmap -h`)

The following options were verified directly from `planarmap.exe -h`:

- Map families:
  - `-C<2,3,4>`: connected cubic map families
  - `-Q<1,2,3>` / `-Q4`: quartic map families
  - `-M<1,2,3>` / `-M4`: connected map families
  - `-B<1,2>`: bipartite cubic map families
- Size/quantity parameters:
  - `-N<nb>`: number of maps generated by PlanarMap
  - `-V<nb>`: number of vertices
  - `-E<nb>`: number of edges
  - `-F<nb>`: number of faces
  - `-R<nb>`, `-G<nb>`, `-I<nb>`: specialized sizing controls
- Output controls:
  - `-d`: output dual map
  - `-p`: print map
  - `-O<num>`: output format (`3` = Maple)
- Reproducibility/system:
  - `-X<seed>`: random seed
  - `-h`, `-v`, `-w`: help/verbosity flags

For this Python wrapper, prefer:
- `--count` to control how many samples the wrapper generates
- PlanarMap flags (`-V`, `-F`, `-C*`, `-M*`, `-Q*`, `-d`, etc.) to control map type and size

In dual mode (`-d`) with cubic maps (`-C2`):
- `-F<n>` controls triangulation vertices
- `-V<n>` controls triangulation faces

Important exact-size note (from `planarmap -h`):
- `-I<nb>` means approximate size `[N +/- I]`
- Use `-I0` to force exact size
- For exactly 100 triangulation vertices, use `-F100 -I0 -d`

Validated 100-vertex triangulation example:

```bash
python batch_dcel_state_generator.py --count 1 -C3 -F100 -I0 -d
```

### Quick Recipes (Copy/Paste)

If you are new to PlanarMap, use these commands directly.

Set PlanarMap path in PowerShell:

```powershell
$env:PLANARMAP_BIN = "Path\PlanarMap-v1-2b\planarmap.exe"
```

Generate 1 DCEL state file with 100 vertices:

```bash
python batch_dcel_state_generator.py --count 1 -C3 -F100 -I0 -d
```

Generate 100 DCEL state files with 100 vertices each:

```bash
python batch_dcel_state_generator.py --count 100 -C3 -F100 -I0 -d
```

Generate 500 DCEL state files with 80 vertices each:

```bash
python batch_dcel_state_generator.py --count 500 -C3 -F80 -I0 -d
```

Use a fixed seed for reproducible runs:

```bash
python batch_dcel_state_generator.py --count 50 --seed 12345 -C3 -F100 -I0 -d
```

View one generated state with the modular unfolding viewer (`--unfold`):

```bash
python "../Example Scripts/UnfoldingProject/coin_unfolding_modular.py" --unfold ./state_output/triangulation_0000_map0_state.json --method bfs
```

See the original PlanarMap-v1-b2 files for a more comprehensive understanding of available flags. 

Save outputs to custom folders:

```bash
python batch_dcel_state_generator.py --count 200 -C2 -F100 -d --output-dir ./maps_json --state-output-dir ./dcel_state_json --test-suite-dir ./MyTestSuite --test-suite-manifest ./MyTestSuite/manifest.json
```

### What You Get After Each Run

- Intermediate PlanarMap JSON in `--output-dir` (default: `./batch_output`)
- koebepy DCEL state JSON in `--state-output-dir` (default: `./state_output`)
- Consolidated test suite copies plus a manifest in `--test-suite-dir` (default: `./TestSuite`)

Each generated DCEL state file is ready to load as koebepy DCEL state JSON.

### Common Mistake

- Do not use `-n` for PlanarMap size. PlanarMap expects flags like `-V100` or `-F100`.
- If you see `Size must be given somehow`, add a size flag such as `-F100`.

**Useful options:**
- `--output-dir`: directory for intermediate map JSON files
- `--state-output-dir`: directory for generated koebepy DCEL state JSON files
- `--test-suite-dir`: final organized test suite directory
- `--skip-json`: reuse existing map JSON files (only convert to state)
- `--skip-state`: skip state JSON generation (only generate map JSON)

Example: reuse existing JSON and only convert to state:
```bash
python batch_dcel_state_generator.py --skip-json --output-dir ./batch_output --state-output-dir ./state_output
```

### Programmatic Use

```python
from koebepy_dcel_state_exporter import KoebepyDCELExporter

exporter = KoebepyDCELExporter()
state = exporter.dcel_json_to_state(vertices, halfedges, faces)
exporter.save_dcel_state(vertices, halfedges, faces, 'triangulation_0000_state.json')
```

## Core Modules

### `triangulation_pipeline_scaffold.py`
- Reads PlanarMap output or fixture triangulation
- Builds DCEL topology from coordinates and triangles
- Emits DCEL topology in JSON format
- Functions: `run_planarmap_capture()`, `convert_planarmap_text_to_json()`, `build_dcel_from_triangulation()`

### `koebepy_dcel_state_exporter.py`
- Converts DCEL topology into koebepy state format
- **Key methods:**
  - `KoebepyDCELExporter.dcel_json_to_state()` â€“ converts data structures
  - `KoebepyDCELExporter.save_dcel_state()` â€“ writes to file
  - `export_dcel_state_from_json_file()` â€“ file-to-file conversion

### `batch_dcel_state_generator.py`
- Orchestrates full pipeline: run planarmap â†’ parse â†’ convert â†’ organize
- **Key functions:**
  - `run_planarmap_batch()` â€“ generate multiple triangulations
  - `json_to_states()` â€“ batch conversion to koebepy state format
  - `organize_test_suite()` â€“ structure output as test suite

### `planarmap_config.py`
- Configuration utility for locating PlanarMap executable
- Respects environment variables and fallback paths

### `test_pipeline_integration.py`
- Integration tests for the full pipeline
- Validates imports, single export, batch API, and examples
- Run tests: `python test_pipeline_integration.py`
